# The `aim-antigravity` Reincarnation Protocol

When you (the operator) type the specific command:
`/reincarnate "Read your HANDOFF.md and follow the directions"`

You are actively firing the core workflow trigger perfectly mapped inside `.agents/workflows/reincarnate.md`. Here is the exact, step-by-step breakdown of what happens biologically under the hood.

---

## 1. The Trigger Phase
The active agent detects the `/reincarnate` slash command. This initiates an immediate "Self-Termination Protocol". It knows it must stop writing code, stop searching databases, and immediately summarize its existence to hand the relay baton cleanly to the next instance.

## 2. Extracting the Signal (The Flight Recorder)
Before the agent dies, it natively uses its LLM reasoning chain to reflect on *everything* it has done in the current chat session. 
It writes a high-fidelity markdown log directly to `.agents/continuity/LAST_SESSION_FLIGHT_RECORDER.md`. This captures the exact technical delta of what was accomplished, what files were touched, and what was discussed.

## 3. Determining the Project Edge (The Pulse)
The agent synthesizes the raw chronological flight recorder into a state-driven "Pulse." It determines exactly what task was finished, what is currently fundamentally broken, and what explicitly needs to happen next. It writes this tightly coupled summary to `.agents/continuity/CURRENT_PULSE.md`.

## 4. Synthesizing the Commander's Intent (The Gameplan)
You provided an explicit parameter in your slash command: **"Read your HANDOFF.md and follow the directions."**
The active agent combines this literal string with its own technical knowledge (from the Pulse) and writes a highly prescriptive, 3-step numbered battle plan to `.agents/continuity/REINCARNATION_GAMEPLAN.md`.

In this specific scenario, Step 1 of the Gameplan generated by the active agent will literally be: *"Action: Read the `HANDOFF.md` file located in the root directory to receive the human's explicit directions."*

## 5. The Splice (The Human Action)
The agent prints out a final message telling you to terminate its existence. 
You, the human, close the visual IDE chat window, open a completely fresh Antigravity session, and boot up the next agent instance.

## 6. Waking Up (The Core Directive Inheritance)
The fresh agent spawns with an entirely blank short-term memory. However, because we mapped `.agents/workflows/core_directives.md` universally to all incoming agents, its highest priority mandate is explicitly defined:
> *"Before you begin any new tactical work, you must read the following files to inherit epistemic certainty: 1. REINCARNATION_GAMEPLAN.md"*

The new agent autonomously executes a `view_file` on `REINCARNATION_GAMEPLAN.md`. It sees the strict mandate from its predecessor: *"Read HANDOFF.md"*. 
It immediately executes a `view_file` on `HANDOFF.md`, learns the new goal, and flawlessly continues the software project exactly where you left off!

---
**Summary:** The complex Python window-scraping scripts are entirely gone. The IDE relies on an unbreakable chain of text documents constantly communicating back and forth through the agent's core memory mandates!
